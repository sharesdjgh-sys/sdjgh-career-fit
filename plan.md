# 진로나침반 — 고등학생 맞춤형 AI 진로 상담 서비스 기획서

> **직업백과 579개 직업 데이터 × 커리어넷 진로심리검사 × Claude AI**
> 작성: 개발4팀 | 2026년 6월 | 본 문서는 `진로상담_AI서비스_기획서.docx`(2026.05)를 기반으로 MVP 확정 사항을 반영해 갱신한 버전입니다.

---

## 1. 서비스 개요

학생의 진로심리검사 결과지를 기반으로 개인맞춤 직업을 추천하고, **"이 직업을 가지려면 무엇을 공부해야 하는가"** 를 AI 분석 보고서 형태로 제공하는 웹 서비스.

| 항목 | 내용 |
|------|------|
| 서비스명 | 진로나침반 (가칭) |
| 대상 | 고등학생 1~3학년 |
| 목적 | 검사 결과지 사진 또는 자유 입력 → AI 분석 → 직업·전공·학습 경로 추천 보고서 |
| 데이터 | 직업백과 579개 직업 (11개 분야), Holland 코드·계열·고교과목·적성요인 태깅 완료 |
| 활용 검사 | 커리어넷 직업흥미검사(Holland RIASEC), 직업적성검사(10개 요인), 직업가치관검사 |
| 기술 스택 | Next.js 단일 풀스택 앱 + Claude API, Vercel 배포 |
| 개인정보 | 서버 무저장 — 학생 데이터는 브라우저(localStorage)에만 보관 |

### 1.1 배경 및 필요성

- 고등학생의 진로 결정 시기가 빨라지고 있으나, 학교 내 진로상담 인력은 한정적
- 커리어넷 적성검사 활용률은 높으나, **결과를 직업·전공·과목과 구체적으로 연결해 주는 도구 부재**
- 보유한 직업백과 579개 데이터를 AI와 결합하면 대규모 맞춤 상담 자동화 가능
- 학생이 스스로 탐색하고, 교사는 심층 상담에 집중하는 분업 모델 구현

### 1.2 서비스 목표

- **정량**: 연간 학생 1인당 최소 1회 이상 진로 탐색 경험 제공
- **정성**: 직업–전공–과목 연계 이해도 향상, 진로 불안 해소
- **운영**: 교사의 기초 진로상담 업무 부담 30% 이상 경감

## 2. 사용자 시나리오

### 시나리오 A — 검사 결과지 사진 업로드 (핵심 경로)

> 고2 김민준 학생은 커리어넷 직업흥미검사에서 탐구형(I)–현실형(R) 결과를 받았지만, 어떤 직업·학과를 선택해야 할지 막막하다.

1. 서비스 접속 → **결과지 사진 촬영/업로드** (휴대폰 카메라 또는 파일)
2. 선택 입력: **MBTI**, 학년, 계열(문·이과), 기타 하고 싶은 말
3. Claude Vision이 결과지에서 Holland 유형·점수, 적성 요인, 가치관 순위를 자동 추출
4. AI가 579개 직업과 매칭 → **Top 10 직업 + 매칭도 + 추천 이유** 제시
5. 관심 직업 선택 → 상세 정보(연봉·전망·전공·고교과목) 탐색
6. **AI 분석 보고서 생성**: 추천 직업별 학습 가이드, 고1~3 로드맵, 수능 전략
7. AI 상담 채팅으로 자유 질문 → 보고서 인쇄/PDF 저장 후 진로상담 교사와 공유

### 시나리오 B — 결과지 없이 자유 입력

> 고1 이수아 학생은 아직 검사를 받지 않았다. '사람을 돕는 일'에 관심이 있고 생물·국어를 좋아한다.

1. **간편 입력 모드**: 관심 분야·좋아하는 과목·성향·중요 가치 카드 선택 (5문항)
2. **자유 서술**: 좋아하는 것, 해본 활동·경험, 관심사를 자유롭게 입력
3. AI가 입력 내용으로 Holland 유형과 적성을 추정 → 최적 직업 추천
4. 이후 흐름은 시나리오 A와 동일 (+ "나중에 커리어넷 검사도 받아보세요" 안내)

### 4.1 보유 데이터

| 데이터 | 규모 | 내용 |
|--------|------|------|

### 4.2 서비스 데이터 파이프라인

```
jobs/**/*.md ──(build-content.mjs)──► data/content/{id}.md   # 직업 본문 (상세 페이지·AI 근거)
jobs.json    ──(이동)──────────────► data/jobs.json          # 매칭 인덱스 (서버 전용)
```

- 서버에서만 로드 (클라이언트로 790KB 전송 금지) — 화면에는 필요한 추천 결과만 전달
- 직업 상세 579페이지는 빌드 시 정적 생성(SSG)
- 학생 프로필·추천 결과·보고서는 **localStorage에만 저장** (서버 DB 없음)

### 4.3 학생 프로필 모델

| 필드 | 내용 | 출처 |
|------|------|------|
| holland | 주·부 유형 + 점수(0~100) | 결과지 추출 또는 성향 문항 추정 |
| aptitudes | 상위 적성 요인 3~4개 (언어력·수리력·추리력 등 10요인) | 결과지 추출 또는 과목·서술 추정 |
| values | 가치관 우선순위 (안정성·보수·성취·워라밸 등) | 결과지 또는 가치 카드 |
| interestCategories | 관심 분야 (11개 카테고리) | 카드 선택 |
| favoriteSubjects / freeText | 좋아하는 과목 / 자유 서술 | 직접 입력 |
| mbti (선택) | MBTI 4글자 | 직접 입력 |
| grade / track | 학년 / 계열(문·이과·예체능·미정) | 직접 입력 |

## 5. 직업 추천 알고리즘

**1단계 — 룰 기반 점수화 (전체 579개, 만점 100점)**

| 블록 | 배점 | 기준 |
|------|------|------|
| Holland 일치 | 40 | 주↔주 25, 주↔부 15, 부 유형 교차 10·8 |
| 적성 요인 | 20 | 학생 상위 요인 ∩ 직업 요구 요인, 개당 5점 |
| 가치관 | 20 | 상위 3개 가중 10/6/4 × 지표 매핑(안정성→고용안정성, 보수→연봉, 발전→발전가능성, 워라밸→일가정균형) |
| 관심 분야 | 10 | 선택 카테고리 일치 |
| 계열·과목 | 10 | 계열 일치 5 + 좋아하는 과목↔고교 추천과목 겹침 5 |

- '정보없음' 지표는 중립(0.4) 처리해 신직업이 부당하게 밀리지 않도록 함
- 카테고리당 최대 ~8개 다양성 가드로 특정 분야 쏠림 방지

**2단계 — Claude 재랭킹**: 상위 30개 후보의 메타데이터를 Claude에 전달 → Top 10 선정, 매칭도(%)·추천 이유 생성. 후보 목록 밖 직업 추천 불가(허용목록 검증), 매칭도는 룰 점수 ±15 범위로 클램프. **API 실패 시 룰 기반 Top 10으로 자동 폴백** — 추천 기능은 절대 중단되지 않음.

## 6. AI 설계 (Claude API)

| API | 역할 | 모델 | 핵심 설계 |
|-----|------|------|----------|
| `POST /api/analyze` | 결과지 사진 → 프로필 추출 (Vision) / 간편 입력 보정 | claude-sonnet-4-6 | 구조화 출력(JSON Schema, enum 제약)으로 Holland·적성·가치관만 허용. 결과지가 아니거나 판독 불가 시 `parse_ok:false` → 간편 입력 유도 |
| `POST /api/recommend` | 후보 30 → Top 10 재랭킹 + 추천 이유 | claude-sonnet-4-6 | 제공된 후보 데이터만 근거로 작성(환각 방지), 실패 시 결정적 폴백 |
| `POST /api/report` | AI 분석 보고서 생성 | claude-sonnet-4-6 | 선택 직업(기본 3개)의 **직업백과 본문 전문을 근거로** 학습 가이드·로드맵·수능 전략 생성. 본문에 없는 자격증·기관명 생성 금지 |
| `POST /api/chat` | 진로 상담 스트리밍 채팅 | claude-sonnet-4-6 | 프로필 + 현재 직업 본문을 시스템 프롬프트에 유지, prompt caching으로 멀티턴 비용 절감 |

- **비용 추정**: 학생 1세션(분석+추천+보고서+채팅 5턴) ≈ **$0.20~0.35**. 절감 필요 시 일부 단계를 claude-haiku-4-5로 전환 시 ~$0.12
- **안전장치**: API 키는 서버 환경변수로만 관리, 호출별 max_tokens 상한, 오류 시 한국어 안내 + 폴백 경로

| 4. 추천 결과 | `/results` | Holland 요약 카드, 분야 필터·정렬, 직업 카드 그리드(매칭도·태그·추천 이유) |
| 5. 직업 상세 | `/jobs/[id]` | 핵심 지표 4카드, Holland 차트, 하는 일·전망, 추천 전공·고교과목, 관련 직업 칩 |

| 8. AI 보고서 | `/report` | 프로필 요약, Top 직업 + 학습 가이드, 로드맵, 수능 전략, 교사 메모란, **인쇄/PDF(A4)** |

**사용자 흐름**: 입력(2 또는 3) → AI 분석 → 추천 결과(4) → 상세(5)/전공(6) 탐색 → 상담(7) ↔ 보고서(8) 저장·인쇄

## 8. 기술 아키텍처

| 영역 | 선택 | 비고 |
|------|------|------|
| 프레임워크 | Next.js (App Router, TypeScript, Tailwind v4) | 프론트+API 단일 앱 — 별도 백엔드·벡터DB·RDB 없음 |
| AI | Claude API (@anthropic-ai/sdk) | 서버 API 라우트에서만 호출 |
| 데이터 | 파일 기반 (data/jobs.json + data/content/*.md) | 서버 메모리 캐시, 579개 규모에 최적 |
| 상태 | localStorage (프로필·추천·보고서), sessionStorage (채팅) | 서버 무저장 = 개인정보 리스크 최소화 |
| 배포 | Vercel | 환경변수 `ANTHROPIC_API_KEY` |
| 인쇄/PDF | `@media print` + 브라우저 인쇄 | A4 레이아웃, 흑백 인쇄 대응 |

### 보안·개인정보

- 학생 식별정보(이름·학번) 미수집, 결과지 이미지는 분석 후 즉시 폐기(서버 미저장)
- API 키 서버 전용(`NEXT_PUBLIC_` 금지), 호출량 제한 + max_tokens 상한

| 결과지 판독 실패 | 사진 흐림, 다른 검사지, 형식 변경 | `parse_ok:false` 감지 → 간편 입력 모드로 자동 유도 (입력값 이관, 막다른 길 없음) |

| 데이터 노후화 | 직업 정보 갱신 미반영 | 직업백과 md 갱신 → 빌드 스크립트 재실행 프로세스 수립 |
