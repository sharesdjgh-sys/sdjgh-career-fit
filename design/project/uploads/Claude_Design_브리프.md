# 진로나침반 — Claude Design 서비스 브리프

> 이 문서는 Claude Design이 "진로나침반" 서비스를 이해하고 디자인 시안을 만들 수 있도록 작성한 브리프입니다.
> 서비스의 실제 구현(2026-06 기준)을 그대로 반영했으므로, 여기 적힌 화면 구성·콘텐츠·상태를 기준으로 디자인해 주세요.
> ※ 기존 `Claude_Design_프롬프트.md`(파란색 팔레트, 슬라이더 입력 화면)는 폐기된 초기 기획입니다. 이 문서가 최신입니다.

---

## 1. 서비스 한 줄 정의

**고등학생이 진로심리검사 결과지 사진 한 장(또는 5문항 간편 입력)으로, 579개 직업 중 나에게 맞는 직업·전공·고교 선택과목을 AI에게 추천받고, 인쇄 가능한 진로 분석 보고서까지 받는 웹 서비스.**

- 운영 주체: 교육청 (학교 현장에서 진로 수업·상담에 활용)
- 데이터: 직업백과 579개 직업 × 11개 분야, Holland 직업흥미 6유형(R·I·A·S·E·C) 기반 매칭
- AI: Claude API — 결과지 사진 판독(Vision), 직업 재랭킹, 보고서 생성, 스트리밍 상담 채팅

## 2. 사용자

| 사용자 | 상황 | 니즈 |
|---|---|---|
| **고등학생 (주 사용자, 여고생 비중 높음)** | 진로 수업 시간 또는 집에서 모바일/PC로 접속 | "내가 뭘 하면 좋을지"를 예쁘고 재미있게, 그러나 진지하게 알고 싶음 |
| 교사 (진로·담임) | 수업 진행, 상담 자료로 보고서 인쇄 | 신뢰할 수 있는 근거, A4로 깔끔하게 출력되는 보고서 |
| 학부모 | 자녀가 받아온 보고서 열람 | 공신력, 읽기 쉬운 정리 |

**톤 앤 보이스**: 전 화면 한국어, 부드러운 존댓말("~해요", "~드려요"). 학생에게는 친근하게, 결과물(보고서)은 공문서 수준의 격식.

## 3. 디자인 히스토리와 이번 목표 (중요)

| 버전 | 방향 | 평가 |
|---|---|---|
| v1.0 | 핑크/바이올렛 그라디언트, 글래스모피즘, 오로라 블러, 알약 버튼 | 화려하지만 유치함. 신뢰감 부족 |
| v2.0 (현재) | 뮤트 로즈(#E85B88) + 그레이 퍼플(#7C6FA8), Linear/Stripe식 절제된 카드·토큰 시스템 | 정돈됐지만 **무난하고 평범함. 기억에 남지 않음** |

**v3 목표: "절제됐지만 개성 있는" 디자인.**
- v1의 실패(과한 장식, 유치함)와 v2의 실패(개성 없는 범용 SaaS 룩)를 모두 피할 것
- 10대 학생이 "오 예쁘다, 캡처해서 공유하고 싶다"고 느끼는 감성 + 교사·학부모가 신뢰하는 격
- "진로를 찾는 나침반"이라는 브랜드 모티프(방향, 탐색, 지도, 여정)를 시각 언어로 살리는 것 환영
- 컬러·타이포·일러스트레이션·모티프는 자유롭게 재제안 가능. 단, 교육 서비스라는 맥락과 한국어 조판을 지킬 것

## 4. 기술·디자인 제약

- **폰트**: Pretendard (한국어 최적화 산세리프). `word-break: keep-all` 필수
- **구현 스택**: Next.js + Tailwind CSS v4 (`@theme` 토큰 방식) — 시안은 자유롭게 만들되, 토큰화 가능한 일관된 시스템이면 이식이 쉬움
- **아이콘**: lucide-react (스트로크 아이콘). 이모지는 UI에 사용하지 않음
- **라이트 모드 전용** (다크모드 없음)
- **반응형**: 모바일(학생 폰) 우선 + 데스크톱(교실 PC). 본문 폭 max-w-5xl 수준
- **/report 화면은 A4 인쇄가 1순위 요구사항** — 흑백 인쇄에도 위계가 살아야 하고, 페이지 나눔이 자연스러워야 함
- AI 생성 텍스트는 마크다운(굵게, 목록)으로 렌더링됨 — 본문 타이포 스타일이 마크다운 요소(strong, ul, table)를 커버해야 함

## 5. 전체 사용자 흐름

```
홈(/)
 ├─ [결과지 사진으로 시작] → 사진 업로드(/input/photo) ─┐
 └─ [간편 입력으로 시작]   → 간편 입력(/input/quick)  ─┤
                                                      ▼
                              AI 분석 → 추천 결과(/results)
                                          ├─ 직업 상세(/jobs/[id]) ↔ 전공·로드맵(/majors)
                                          ├─ AI 상담 채팅(/chat)
                                          └─ 분석 보고서(/report) → 인쇄/PDF
```

상단 글로벌 헤더(전 화면 공통): 로고(나침반 아이콘 + "진로나침반") / 네비: 추천 결과 · AI 상담 · 보고서. 푸터: 한 줄 크레딧.

## 6. 화면별 상세

### 6-1. 홈 `/` — 랜딩

서비스 첫인상. 구성 요소:
1. **히어로**: 배지(“AI 기반 맞춤 진로 가이드”), 헤드라인 “나만의 진로를 찾아보세요”, 서브카피(“진로심리검사 결과지 사진 한 장이면 충분해요…”), CTA 2개 — **[결과지 사진으로 시작하기]**(주), **[간편 입력으로 시작하기]**(보조), 보조 안내문(“검사를 아직 안 받았어도 괜찮아요”)
2. **특징 카드 3장**: 맞춤 직업 추천(579개 중) / 전공·학과 안내 / 고교 과목 가이드 — 아이콘 + 제목 + 2줄 설명
3. **통계 배너**: 579개 직업 데이터 · 11개 직업 분야 · AI 맞춤 분석 보고서
4. **이용 순서 4단계**: ① 결과지 업로드 또는 간편 입력 → ② AI가 흥미·적성 분석 → ③ 맞춤 직업 Top 10 추천 → ④ 학습 로드맵 보고서

### 6-2. 사진 업로드 `/input/photo`

커리어넷 진로심리검사 결과지 사진을 올리는 화면.
1. **드롭존**: 드래그앤드롭 + 파일 선택. 업로드 후 썸네일 미리보기, 드래그 중 강조 상태
2. **추가 정보(선택 입력)**: 학년 칩(고1/고2/고3), 계열 칩(문과/이과/아직 모름), MBTI 텍스트 입력(4글자, 형식 검증 에러 있음), 자유 서술 텍스트영역(“좋아하는 것·잘하는 것을 자유롭게”)
3. **제출 CTA**: “AI 분석 시작하기”. 상태: 로딩(스피너 + 분석 중 문구), 에러 박스(사진 판독 실패 등)

### 6-3. 간편 입력 `/input/quick`

결과지가 없는 학생용. **단계형 카드 + 상단 진행 바** 구조, 5단계:
1. 관심 분야 (복수 선택 칩, 11개 분야: IT·디지털기술 / 의료·보건 / 교육 / 예술·문화·미디어 / 경제·금융·법률 / 공학·제조·건설 / 환경·에너지·농업 / 공공·사회·복지 / 서비스·유통·음식 / 연구·과학 / 스포츠·레저)
2. 좋아하는 과목 (복수 선택 칩: 수학/과학/국어/영어/사회/역사/미술/음악/체육/정보 등 12개 내외)
3. 나의 성향 (단일 선택 카드 5개: 혼자 연구 / 사람들과 협업 / 창작 / 정리·분석 / 리드·설득)
4. 중요 가치 (최대 2개: 높은 연봉 / 안정성 / 사회 기여 / 창의성 / 성장 가능성 / 워라밸)
5. 학년·계열 + MBTI(선택) + 자유 서술(선택)
- 하단 [이전]/[다음], 마지막 단계 [결과 보기] 제출(로딩 스피너). 칩이 많아 모바일 밀도·터치 타깃이 중요

### 6-4. 추천 결과 `/results`

분석 결과의 메인 화면.
1. **프로필 요약 카드**: “OO 유형의 학생이에요” — Holland 주·부유형 배지(예: 탐구형 I · 사회형 S), 학년·계열·MBTI 메타, [다시 입력] 링크. 간편입력 기반이면 “추정 결과” 경고 박스
2. **필터/정렬 바**: 분야 필터 탭(전체/IT·디지털기술/의료·보건/…), 정렬 셀렉트(매칭도순/연봉순/발전가능성순)
3. **직업 카드 리스트 (Top 10)**: 카드당 — 분야 아이콘+분야명(메타), **직업명(주인공)**, **매칭도 %**(90% 이상 강조), AI 추천 이유 1~2문장, 배지(계열·연봉수준·발전가능성·고용안정), [자세히 보기]
4. **하단 고정 액션 바**: [AI 상담사에게 물어보기] [보고서 만들기]
- 빈 상태: “아직 분석 결과가 없어요” + 시작 CTA

### 6-5. 직업 상세 `/jobs/[id]` — 579개 정적 페이지

1. **히어로**: 분야 배지, 계열 배지, (추천받은 직업이면) 매칭도 배지, **직업명 크게**, 직무 한 줄, 해시태그 칩들
2. **핵심 지표 카드 4개**: 평균연봉 / 발전가능성 / 고용안정성 / 일·가정 균형 (값은 “높음/보통/낮음” 또는 금액)
3. **직업 만족도**: % + 프로그레스 바
4. **Holland 성향 매칭**: 해당 직업의 유형 1~2개 카드(유형명 + 설명) + 핵심 적성 요인
5. **추천 전공/고교 과목**: 칩 리스트 + “3개년 로드맵 보기” 링크
6. **직업백과 본문**: 마크다운 장문(“## 하는 일”, “## 필요한 능력”, “## 전망” 등 — 소제목·문단·표·목록 포함). 가독성 최우선
7. **관련 직업 칩** (클릭 시 해당 직업으로 이동)
8. **하단 고정 액션 바**: [목록으로] [이 직업 상담하기] [전공 추천 보기]

### 6-6. 전공·로드맵 `/majors`

선택한 직업 기준 진학 가이드.
1. 상단: “OO(직업명)를 위한 전공 가이드”
2. **추천 학과 카드들**: 학과명 + 계열 배지 + 관련 과목 칩
3. **고교 3개년 로드맵 타임라인**: 고1 → 고2 → 고3 세로 스텝, 학년별 추천 선택과목·활동
4. **수능 전략 박스**: 권장 수능 선택과목 조합
5. 하단 고정 액션 바: [직업 상세] [AI 상담] [보고서 만들기]

### 6-7. AI 상담 채팅 `/chat`

1. **좌측 사이드 패널(데스크톱만)**: 내 프로필 카드(Holland 배지, 학년·계열·MBTI, 추천 직업 Top 3 리스트 — 클릭 시 해당 직업 컨텍스트로 전환), 자주 묻는 질문 버튼 5개(클릭 시 입력창에 자동 입력)
2. **채팅 영역**: 상단 바(“진로 탐색 AI 상담” + 온라인 점 + 현재 상담 중인 직업 배지), 말풍선(학생: 우측 강조색 / AI: 좌측 흰 카드 + 나침반 아바타), AI 응답은 **스트리밍**(타이핑 점 3개 로딩), 마크다운 렌더링(굵게·목록)
3. **하단 입력창**: 텍스트 인풋 + 전송 버튼(비활성 상태 있음)
- 높이는 화면 고정(100vh-헤더), 채팅 영역만 스크롤

### 6-8. 분석 보고서 `/report` — ★ A4 인쇄 1순위

학생이 교사·부모에게 제출하는 공식 문서. 화면용과 인쇄용을 모두 만족해야 함.
1. **문서 헤더**: 로고 + “진로나침반 AI 리포트”, 발행일, 제목 “진로 심층 분석 보고서”, **소속 학교·학생 이름 직접 입력란**(밑줄 인풋), 대상 학년
2. **§1 학생 성향·Holland 분석**: AI 요약문 + Holland 6유형 점수 막대 그래프(주·부유형 강조) + 해설
3. **§2 추천 직업 + 학습 가이드** (Top 3~5, 직업당 카드): 순번 + 직업명 + 매칭율 배지, 추천 이유, 학년별 지향 선택과목(고1/고2/고3), 추천 대학 전공, 추천 자격증, 권장 교내외 활동
4. **§3 고교 3개년 학업 로드맵**: 고1→고2→고3 세로 타임라인(현재 학년 강조)
5. **§4 수능 시험 전략** 박스
6. **마무리 조언** (감성적 응원 한 단락)
7. **교사 평가·상담 기록란**: 빈 점선 박스 (인쇄 후 수기 작성용)
8. 푸터: 데이터 출처·면책 문구
9. **액션 버튼(인쇄 시 숨김)**: [리포트 인쇄] [PDF로 저장] [리포트 다시 생성] [결과로 돌아가기]
- 상태: 로딩(“AI 심층 진로 보고서 작성 중” + 스피너), 에러(재시도 버튼), 데이터 없음(시작 유도)

## 7. 공통 컴포넌트 인벤토리

디자인 시스템이 커버해야 하는 요소들:

- **버튼**: Primary / Secondary / Tertiary(텍스트형), 큰 CTA, 비활성, 로딩(스피너)
- **카드**: 기본 정보 카드, 인터랙티브(호버) 카드, 직업 추천 카드, 지표 카드
- **칩/배지**: 선택형 칩(선택·비선택·호버), 필터 탭(활성·비활성), 정보 배지(분야·계열·매칭도·연봉 등), 시맨틱 배지(성공/경고/오류/정보)
- **입력**: 텍스트 인풋, 텍스트영역, 셀렉트, 밑줄형 인풋(보고서), 파일 드롭존, 폼 에러 메시지
- **진행 표시**: 단계 진행 바, 점수 막대 그래프, 프로그레스 바, 타임라인(세로 스텝)
- **상태 화면**: 빈 상태, 로딩(전체 화면), 에러(재시도)
- **채팅**: 말풍선 2종, 타이핑 인디케이터, 아바타
- **본문 타이포**: 마크다운(h2/h3, 문단, 목록, 표, 굵게, 구분선) — 직업백과·AI 텍스트 공용
- **하단 고정 액션 바**: 2~3버튼 조합 (모바일 핵심 동선)
